**Root Cause**: The page is showing **recent transactions (2024/2025)** and hasn't scrolled back to 2023 yet. The extension needs to scroll backward to load older transactions.

**Troubleshooting Steps**:

1. **Check Browser Console (F12)**:
   - Look for date parsing errors
   - Check if scrolling is happening
   - Verify start/end dates are parsed correctly

2. **Verify Date Format**:
   - Input should be: `2023-01-01` to `2023-12-31` (YYYY-MM-DD)
   - Extension will convert to MM/DD/YYYY internally

3. **Check if Scrolling is Working**:
   - The extension should automatically scroll backward to load 2023 transactions
   - If it's not scrolling, check console for errors
   - May need to wait longer (1-year range takes ~15-25 minutes)

4. **Verify Transactions on Page**:
   - The 110 transactions are likely from 2024/2025 (recent)
   - Extension needs to scroll back ~2 years to reach 2023
   - This is normal - scrolling will load older transactions

5. **Wait for Scrolling to Complete**:
   - 1-year range typically takes 15-25 minutes
   - Extension will scroll backward automatically
   - Progress will show in the status bar

**Expected Behavior**:
- Extension should scroll backward from current date (Nov 2025) to Jan 2023
- This takes time (~15-25 minutes for 1 year)
- Transactions will load as scrolling progresses
- Final count should be close to 450 (from 3-year extraction)

**If Still Not Working**:
- Check if "Strict boundaries" is enabled
- Verify date format is YYYY-MM-DD
- Try a shorter range first (e.g., Jan 1 - Mar 31, 2023) to test
- Check console for specific error messages
